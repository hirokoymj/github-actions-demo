Q81: How can you reuse a defined workflow in multiple repositories? (Choose two.)

- By creating a reusable action
- By copying the workflow file to each repository
- 👉 By using workflow templates
- 👉 By defining the workflow in a central repository

💡 https://docs.github.com/en/actions/using-workflows/creating-starter-workflows-for-your-organization

#### 💡 Why these are correct

#### ✔️ Workflow templates

- Stored in the special .github repository
- Used when creating new workflows across repos
- Promotes standardization

#### ✔️ Central repository (reusable workflows)

- You can define a reusable workflow using:

```yaml
on:
  workflow_call:

jobs:
  call-workflow:
    uses: org/repo/.github/workflows/workflow.yml@main
```

Q82: How can you ensure a job runs only on a specific branch?

- By using the branch keyword
- By using the branches filter
- By using the runs-on filter
- By using the jobs filter

💡 https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions#using-filters
✅ Correct answer:

👉 By using the branches filter

🧩 Example

```

  push:
    branches:
      - main
```
