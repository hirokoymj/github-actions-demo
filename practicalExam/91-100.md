Q91: How can you specify the operating system for a job in GitHub Actions?

💡 [doc](https://docs.github.com/en/actions/using-workflows/workflow-syntax-for-github-actions#jobsjob_idruns-on)

✅ Correct answer: 👉 Using the runs-on keyword

#### 💡 Why this is correct

runs-on specifies the runner (operating system/environment) for a job
It tells GitHub where the job should execute

```yaml
jobs:
  build:
    runs-on: ubuntu-latest
```

- ubuntu-latest
- windows-latest
- macos-latest

### 🧠 Key takeaway

OS / runner → runs-on

Q97: How does the actions/cache action in GitHub Actions handle a cache miss?

💡 [doc](https://docs.github.com/en/actions/using-workflows/caching-dependencies-to-speed-up-workflows#managing-caches)

✅ Correct answer:

👉 by automatically creating a new cache if the job is completed successfully

#### 💡 Why this is correct

- When actions/cache does not find a matching cache (cache miss):

1. The workflow continues normally
2. At the end of the job (if successful), GitHub:
   - 📦 creates and saves a new cache using the specified key

#### 🧩 Example flow

```yaml
- uses: actions/cache@v4
  with:
  key: my-cache-key
```

👉 Behavior:

- 🔍 Cache found → restore it
- ❌ Cache miss → run job normally
- ✅ Job succeeds → save new cache

Q100: Select status check functions in GitHub Actions

- status(), always(), cancelled() and failure()
- 👉 success(), always(), cancelled() and failure()
- completed(), always(), cancelled() and failure()
- state(), always(), cancelled() and failure()

💡 [doc](https://docs.github.com/en/actions/learn-github-actions/expressions#status-check-functions)

#### 💡 Why this is correct

GitHub Actions provides these status check functions:

- `success()` → runs if previous steps succeeded
- `failure()` → runs if any previous step failed
- `cancelled()` → runs if the workflow was cancelled
- `always()` → runs regardless of status

#### 🧩 Example

```yaml
steps:
  - run: npm test

  - name: Run on failure
    if: failure()
    run: echo "Tests failed"

  - name: Always run
    if: always()
    run: echo "Cleanup"
```
